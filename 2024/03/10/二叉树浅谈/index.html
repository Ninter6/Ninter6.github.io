前言

这几天突发奇想研究了一下二叉树，发现这东西确实有些能讲的。这篇算是一个笔记，也算记录一下自己的一些疑惑。

树

树是一种数据结构，它是由n(n≥0)个有限节点组成一个具有层次关系的集合。把它叫做“树”是因为它看起来像一棵倒挂的树，也就是说它是根朝上，而叶朝下的。

它具有以下的特点:
- 每个节点有零个或多个子节点；
- 没有父节点的节点称为根节点；
- 每一个非根节点有且只有一个父节点；
- 除了根节点外，每个子节点可以分为多个不相交的子树(Subtree)

补充性质:
- 根节点到达每个节点的路径是唯一的

树的概念不再过多赘述，更多信息请看引用，这里再强调一些重点:
- 度: 子节点的个数
- 深度: 根到此节点的经过的连接数
- 树的高度: 最大深度+1
- 根节点: 没有父亲的节点
- 中间节点: 有父亲有孩子的节点
- 叶子节点: 没有孩子的节点

二叉树

二叉树(Binary Tree)最直接的定义: 度为2的树

二叉树作为一种特殊的树，拥有树的所有性质。

特殊的二叉树

空二叉树(Empty Binary Tree)

没有节点的二叉树。空二叉树的高度是0。

满二叉树(Full Binary Tree)

除了叶子节点都有两个孩子。任一节点的子树都是满二叉树。

只有根节点或着空二叉树也是满二叉树。

性质:
- 如果层序遍历这棵树，那么n号节点的两个孩子必然是(2n+1)和(2n+2)
- 高度为k的满二叉树共有(2^k-1)个节点

完全二叉树(Completed Binary Tree)

一棵深度为k的有n个结点的二叉树，按层序进行编号，如果编号为i（1≤i≤n）的结点与满二叉树中编号为i的结点在二叉树中的位置相同，则这棵二叉树称为完全二叉树。

完全二叉树由于其特殊的性质，所以可以直接存储在数组中。stl中的std::make_heap也是这个原理。

平衡二叉树(Balanced Binary Tree)

任一节点子树的深度差不超过1。

平衡二叉树可以提高搜索效率。

完全二叉树是平衡二叉树。

二叉搜索树(Binary Search Tree)

二叉搜索树中每个节点都满足以下性质:
- 所有左子树的节点的值都小于该节点的值；
- 所有右子树的节点的值都大于该节点的值。

二叉树的遍历

前序遍历(Preorder Traversal)

是一种深度优先遍历方法，会从根节点开始，然后遍历左子树，其次右子树。即遵循"根-左-右"原则。

中序遍历(Inorder Traversal)

首先递归地访问左子树，然后访问根节点，最后访问右子树。即遵循"左-根-右"原则。对于二叉搜索树，中序遍历的结果是按照升序排列的。

后序遍历(Postorder Traversal)

后序遍历先递归地访问左子树，然后访问右子树，最后访问根节点。即遵循"左-右-根"原则。这种遍历方法常用于树的删除操作。

层序遍历(Level Order Traversal)

是一种广度优先遍历方法，从根节点开始，逐层访问树中的每个节点。

代码

对于前三者来说，它们都是基于递归的，所以不可能做进迭代器中。在代码上它们只有访问位置的区别。

而层序遍历是基于队列的。

二叉树的存储

接下来是本文的重点。

二叉树的存储基本可以分为两类:
- 多重链表
- 数组

多重链表

顾名思义，在一个节点中能指向多个节点的链表是多重链表。

一个简单的节点结构可以这么写:

它面临的问题十分明显:
- 难以遍历
- 无法随机读取
- 需要额外管理内存

作为链表，内存分配开销大本身也就是一个问题。
由于不能随机读取，我一般使用时都会用一个哈希表来映射节点以换取速度。
至于内存管理，简单直接的方法就是使用智能指针。当然你也可以选择用引用来构建一棵树，让别的容器来管理。
遍历是最大的问题，如果想开放迭代器，那就只能在迭代器里实现一个BFS。那么迭代器的体积会变得多大就不用说了。

工业上可能会用内存池之类的技术来解决上述问题，但日常使用不可能有这么大的复杂度。

优点:
- 直观
- 增删移动节点容易

当然他也有缺点:
- 增删移动慢的要命
- 如果内存分配很保守，就会产生浪费

这种方法适用于一次性不维护的树，因为不会有增删移动。否则还是使用多重链表。
